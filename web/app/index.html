atlas

Client-side. Your token stays in your browser.

See any GitHub repo as a living map.

Atlas reads a project's specs, source, and git history and draws one interactive picture: a spec and code graph, a coverage treemap and sunburst, a dependency DAG, a churn-vs-coverage view, and more. The whole engine is compiled to WebAssembly and runs here in the page. No server sees your code.

Sign in with GitHub

A short OAuth popup returns a token to this page (and nowhere else). It unlocks your public and private repositories at the full 5,000 requests per hour.

Auth worker not configured

Set workerOrigin in config.js to your deployed auth worker origin (for example https://atlas-auth.your-subdomain.workers.dev), then reload. See web/auth-worker/README.md for the one-time setup.

Try a small public repo first. Atlas fetches the file tree, the source it recognizes, and recent history, then draws the map. Repos with no specs still get a treemap, language mix, and orphan clusters.
